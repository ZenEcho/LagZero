# LagZero 发布检查清单

本文档面向接近发布阶段的自测、联调和打包确认，默认以 **Windows 桌面版** 为主。

## 1. 发布前准备

- 确认当前分支代码、文档和多语言文案都已同步
- 确认 `package.json` 中的版本号符合本次发布计划
- 确认 `pnpm-lock.yaml` 与依赖变更已经提交
- 如果本次改动涉及 deep link、协议注册、扫描器、代理链路或 native 依赖，优先安排完整回归

## 2. 基础命令检查

建议按下面顺序执行：

```bash
pnpm install
pnpm rebuild:native
pnpm build
pnpm test
pnpm pack
pnpm dist:win:all
```

期望结果：

- `pnpm build` 成功结束，无类型错误
- `pnpm test` 成功结束，关键单测无回归
- `pnpm pack` 能产出可运行的未安装目录
- `pnpm dist:win:all` 能产出 Windows 的 NSIS 与 Portable 包

## 3. 产物检查

检查 `release/<version>/` 下是否包含预期文件：

- Windows x64 NSIS 安装包
- Windows arm64 NSIS 安装包
- Windows x64 Portable 包
- Windows arm64 Portable 包

同时确认：

- 文件名包含正确版本号与架构信息
- 启动图标、应用名、安装器名与 `package.json` 中配置一致

## 4. 安装版与便携版都要做的冒烟验证

### 启动与基础界面

- 应用能正常启动并进入主窗口
- 侧边栏路由、窗口控制按钮、主题与语言切换正常
- 托盘图标和托盘窗口可以正常显示
- 关闭按钮行为符合设置中的 `ask / minimize / quit`

### 运行时目录

- 开发态不会误写正式用户目录
- Windows 打包版会在可执行文件同级生成 `data/`
- `data/logs/`、`data/crashDumps/` 路径符合预期
- 首次启动为空目录时，不会因路径初始化失败而崩溃

### sing-box 核心

- 缺少核心时，安装器守卫会正确提示并可触发安装
- 安装完成后能正确识别已安装版本
- 切换首选核心版本后，设置与实际安装状态一致

### 节点与订阅

- 分享链接导入正常
- Clash YAML / Base64 订阅解析正常
- 手动刷新订阅正常
- 同一订阅 URL 重复导入时会更新而不是无限重复新增
- 节点测速、批量测速与排序筛选可正常工作

### 游戏库与加速

- Steam / 本地快捷方式等常用来源扫描正常
- 手动添加游戏、编辑游戏、删除游戏正常
- 启动加速与停止加速正常
- 链式子进程代理在需要时能正常跟进

### 代理与网络

- TUN 模式可正常启动与停止
- 系统代理模式可正确设置和恢复
- 本地代理可自动选点并通过健康检查
- 节点列表变化后，本地代理可自动恢复或切换

### 日志、更新与应用维护

- 版本号显示正确
- 更新检查可以返回结果
- 日志可读取、清理并打开日志目录
- 应用重置后能恢复到预期初始状态

## 5. 自定义协议与网页导入专项检查

至少验证以下链路：

- `lagzero://import?url=<encoded-url>&name=<encoded-name>`
- `clash://install-config?url=<encoded-url>`
- `mihomo://install-config?url=<encoded-url>`

检查点：

- 打包版首次启动后，系统能识别自定义协议
- 二次唤起时会转发给已运行实例，而不是打开第二个实例
- deep link 中 `url` 非 `http/https` 时会被拒绝
- 导入完成后会正确创建或更新订阅
- `clash://`、`mihomo://` 被其他客户端抢占后，协议守护逻辑能按设置恢复

## 6. 回归重点

以下场景是高风险区，发版前尽量全走一遍：

- 原生依赖重建后能否稳定启动
- 打包版路径切换和旧数据迁移
- 本地代理与游戏加速共用 sing-box 进程时的状态同步
- 协议注册、协议守护与单实例 deep link 转发
- 订阅解析兼容性，尤其是 Clash YAML 和 Base64 内容
